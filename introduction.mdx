## Performance

Velo uses a hybrid synchronous and asynchronous model to keep your application fast under heavy load.

When you log a message, Velo formats it on the caller's goroutine and sends it to a buffered channel. A background worker reads from this channel and writes to the output stream. This parallelizes the work and isolates your application code from I/O latency.

## Backpressure handling

In high throughput scenarios, your application might generate logs faster than the output stream can write them. When the buffer is full, backpressure occurs.

You can control how Velo handles a full buffer using the `OverflowStrategy` option:

<AccordionGroup>
  <Accordion title="OverflowSync (Default)">
    The logger temporarily switches to a synchronous write. The calling goroutine writes its preformatted log entry directly to the output stream. This prevents log loss and controls memory use, but temporarily blocks the calling goroutine.
  </Accordion>
  
  <Accordion title="OverflowDrop">
    The logger discards new log entries until space opens up in the buffer. Use this strategy when maintaining low latency is more critical than keeping every log entry.
  </Accordion>
  
  <Accordion title="OverflowBlock">
    The calling goroutine waits and blocks until space becomes available in the buffer.
  </Accordion>
</AccordionGroup>
